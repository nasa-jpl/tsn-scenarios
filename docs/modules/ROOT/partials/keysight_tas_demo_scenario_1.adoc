The following steps configure the Keysight to implement scenario 1 of Time Aware Shaping.  

These steps start with the Keysight configuration implementing basic time synchronization found at xref:802.1AS/keysight.adoc[]

PTP frames are not allowed to include VLAN tags, but for our scheduled and prioritized traffic we need to include that.  Therefore, we need to add a new Ethernet protocol stack, with VLAN enabled, to the port that is acting as the time transmitter.  We will use this to generate scheduled traffic.  Alternatively, one could add a second device group to the same port.

For the demonstration we want some traffic coming from 2 different ports, so we need to add a new port (topology).  This will be used to generate both prioritized but un-scheduled traffic, and vanilla Ethernet traffic.  Vanilla Ethernet traffic will have no VLAN / priority tag, so we need to create 2 Ethernet stacks on this device, one with VLAN enabled.  Since this port is generating un-scheduled traffic, it doesn't need to be synchronized to the network time.  We are not going to setup 802.1AS on this port.  It is not clear, given the same physical device (Keysight) is operating all of these ports, it might get synchronized anyway just because one of the ports is.  It doesn't matter to this scenario, but for more complex scenarios it will be important to use physically separate devices with physically separate oscillators to ensure we are covering more flight-like use cases.

The network definition is now complete.  Next steps are to add the various traffic flows.  A total of 8 separate traffic flows will be added.  

* 4 scheduled traffic, using priorities 1-4.  
** These will generate frames of a fixed size, every 1ms.  
** Initially, these will be configured to output at the same time, so the TAS gating of the switch can be demonstrated.
** Then will setup the Keysight to output them at offsets matching the switch gating to match the scenario.
* 3 priority levels (5-7) of streaming traffic.  This will also be fixed frame sizes, set to output at 10% of line rate.  Keysight will space the frames apart to achieve the desired line rate.
* 1 traffic stream that is not prioritized, representing vanilla ethernet traffic.  This will be set to random packet sizes, at 50% of line rate.
* Have not yet figured out how to rename protocol stacks, so for now you have to pay attention on the Overview screen, which "Ethernet N" to use as source and destination when creating traffic.
