This document provides a general overview of how to configure a TSN switch to support TSN demos.  Specifically, setting up appropriate VLANs to isolate the demo traffic from the rest of the network.

All demos start with this base configuration, and provide detailed instructions for any additional configuration needed specific to that demo.

== Overview of Base Configuration

The base configuration needs to 

* Isolate aanet traffic (the general lab traffic) from the TSN / demo traffic.
** This is accomplished by putting TSN network ports on a non-default VLAN (we will use VLAN 2)
** To keep it simple, devices generating TSN / demo traffic should not use VLAN tagged frames.  Rather, use priority tagged frames.  That would be frames that have a 802.1Q tag but the VLAN ID is 0.  This is a special VID that effectively means no VID is specified.  This allows the talker to specify the priority, and the switch to specify the VLAN.
** To isolate traffic mirrored to a network analyzer, setup a separate VLAN (will use 200) for those ports
* Disable switch background traffic (i.e., LLDP, MSTP) to avoid unexpected traffic.
* Configure a Precision Time Protocol (PTP) instance on VLAN 2
* Disable MAC learning
** Needs to be disabled on network analyzer ports if they are configured to pass all traffic through, or the traffic will create a loop.  For the screenshots shown below, the analyzer was connected to ports 6 and 8.
** For testing, it might be simpler initially to keep MAC learning enabled on other ports, but for flight will most likely disable this so eventually will need to update switch configs to match.
** Prefer disabled on other ports to ensure complete control and match flight-like use-case.  If MAC learning is desired, be sure to set up learning phase on whatever is part of the demo (if something is only a receiver, then the MAC learning cannot happen, Keysight has a way to make all ports send a "no-op" essentially, but have not gotten that to work yet.)
* Enable port classification
** This enables the switch to honor the priority tag
** Our demos generally use just ports 1-4, so enable this on ports 1-4.
* Optional, but disable timeout on the web interface (by default, you have to re-authenticate after 10 minutes of inactivity).  Setting it to 0, disables the timeout.

== Detailed Steps, Microchip TSN Switch

=== Factory Reset

Resetting the switch to factory settings is an optional step but ensures that any modifications made by other users do not impact your configuration.

To follow the instructions to reset and provision the switch, go to the HPSC switch equipment documentation, found https://nasa-jpl.github.io/tsn-scenarios/docs/main/equipment/microchip-sparx-5i-evb/[here].
