This document provides specific instructions for implementing the PSFP scenarios with the Keysight Novus One Plus network emulator acting as the end-points, and the Microchip VSC5640EV switch hardware running iStax software.

It is assumed that there is basic familiarity with the Keysight and switch.  If not, the TAS scenario instructions are much more detailed and provide specific steps for how to create topologies and traffic items in Keysight, etc.

All scenarios assume the base switch configuration has already been performed.  Instructions can be found here: xref:switch-config.adoc[].

All scenarios assume the Keysight is starting with the base 802.1AS time synchronization setup, found here: xref:802.1AS/index.adoc[].

== Detailed Steps, Flow Metering

=== Scenario 1 - Basic CIR / CBS

==== Detailed Switch Configuration

Configure a stream with Stream ID 1. Within the configuration window, set the following settings:

* Set SMAC type to `MAC/Mask`.
* Input the MAC to be the source MAC of the ethernet transmitter on the keysight. The SMAC can be found by clicking on the "Ethernet" protocol box within the Topology in the Keysight, then referencing the "MAC" box in the bottom window.
* DMAC type should remain as `Any`
* Finally, select port 1 (assuming that port 5 of the Keysight is connected ot port 1 of the switch).
* Default remaining settings

Create a flow meter with ID 1:

* CIR = 1000 (1Mbps)
* CBS = 1000000
* EIR = 0
* EBS = 0
* Default remaining settings

To connect the flow meter to the stream, Navigate to TSN -> PSFP -> Stream Filters. Create a stream filter with:

* Stream ID = 1
* Flow Meter Enabled
* Flow Meter ID = 1
* Default remaining settings

==== Detailed Novus Configuration

Create a standard traffic item from the transmitter stack to the receiver. 

For the trafcic item, enable the following settings:

* Under Flow Transmission, click Burst(Custom).
** Set the Duration to be in units of packetCount, with value 1000.
** Enable Inter Burst Gap, with duration 10 seconds.
* Set the frame size to be Fixed: 1000 (Bytes)

This enables 1,000,000 Byte burst traffic (1,000 packets of 1,000 bytes = 1,000,000 bytes).

==== Running the Scenario & Expected Results

===== Test 1

Generate, Apply, and Run the Novus configuration. Either in the switch Port Statistics Overview, or in the Keysight statistics overview, view the Tx and Rx byte count.

Every 10 seconds the transmitted/received bytes should tick up by 1,000,000. No frames should be dropped, as the bit rate fits within the flow meter's bucket. We are sending it at an average 0.8 Mbps, but filling the bucket with tokens at 1 Mbps, so we should never run out of tokens. Therefore, we should see equal amounts of transmit and recevied bytes.

===== Test 2

Now, in the keysight, change the burst spacing to be every 7 seconds, and rerun the scenario.

The first burst should be fully transmitted and received as the bit rate fits within the flow meter's bucket. However, with each successive burst, the flow will be metered to stay under the 1,000,000 byte bucket size. The expected result should see frames dropped from the burst but not all frames. The data rate is approximately 1.14 Mbps, so we should see an oversaturation of the flow meter bucket, and only receive 1 Mbps out.

To see a more drastic effect, reduce the inter burst gap to <5 seconds.
