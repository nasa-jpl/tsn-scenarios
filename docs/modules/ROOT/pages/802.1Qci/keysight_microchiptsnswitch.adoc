This document provides specific instructions for implementing the PSFP scenarios with the Keysight Novus One Plus network emulator acting as the end-points, and the Microchip VSC5640EV switch hardware running iStax software.

It is assumed that there is basic familiarity with the Keysight and switch.  If not, the TAS scenario instructions are much more detailed and provide specific steps for how to create topologies and traffic items in Keysight, etc.

All scenarios assume the base switch configuration has already been performed.  Instructions can be found here: xref:base_switch_config.adoc[].

All scenarios assume the Keysight is starting with the base 802.1AS time synchronization setup, found here: xref:802.1AS/802.1AS.adoc[].

== Detailed Steps, Stream Mapping
=== Scenario 1
Rule Order-Of-Execution

==== Detailed Switch Configuration

*It is critical that the streams be created with the IDs shown and in the order listed*

* Create stream ID 10 that matches everything on EP~1~
* Create stream ID 5 that matches everything on EP~1~
* Create stream ID 20 that matches everything on EP~1~
* Create 3 flow meters
** Use rates CIR of 100, 200, and 400 Kbps.  To make things easy to keep straight, use flow meter IDs 1, 2, and 4 for those.
** Use CBS of 8192 for all 3
* Create 3 stream filters, also with IDs 1, 2, and 4, with the following Stream ID / flow meter / CIR mappings
** Stream ID 5: Flow Meter 1: 100 Kbps
** Stream ID 3: Flow Meter 2: 200 Kbps
** Stream ID 7: Flow Meter 4: 400 Kbps

==== Detailed Novus Configuration

* Create a single traffic flow
** EP~1~ to EP~3~
** 1% line rate
** fixed 64 byte frames

==== Test Execution and Results

The expectation is that the traffic item will only see 200 Kbps get through, as the switch should be checking the frame against stream rules in ascending numeric order, regardless of the order in which they were created.

The results do match that as shown in the screenshots below.  Note that the bit rate being received slightly exceeds 200 Kbps.  It is not understood why that is.
