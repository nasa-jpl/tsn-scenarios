= Ansible

This directory contains a set of Ansible playbooks and roles for executing the scenarios for the Linux endsystems in the COTS venue.

== Dependencies

* Ansible
* Systemd

== Setup

[source,sh]
.Modify inventory as necessary (see `USER` comments)
----
vim inventory
----

[source,sh]
.Setup logging
----
ansible-playbook logging.yml
----

=== Test Logging

[source,sh]
.From one of the DUTs, test logging by sending a message to the log
----
systemd-cat echo test
----

[source,sh]
.On the logging server, check the log
----
sudo journalctl --directory /var/log/journal/remote --merge -n1
----

[source,console]
.Expected output
----
user@logging-server:~$ sudo journalctl --directory /var/log/journal/remote --merge -n1
Jun 09 17:08:13 hpsctsnes1 echo[39637]: test
----

See <<troubleshooting-logging>> if this doesn't work.

== 802.1AS Example

On the logging server, start following the logs:

[source,sh]
----
sudo journalctl --file /var/log/journal/remote/all.journal -mf _COMM=ptp4l
----

In another terminal, start the 802.1AS scenario:

[source,sh]
----
ansible-playbook AS-start.yml
----

When done, stop the 802.1AS scenario:

[source,sh]
----
ansible-playbook AS-stop.yml
----

== Troubleshooting Logging

[source,sh]
.Check that server is responding
----
curl -v http://$logging_server:19532
----

[source,console]
.Expected response
----
ansible@hpsctsnes1:~$ curl http://192.168.1.60:19532 -vv
*   Trying 192.168.1.60:19532...
* Connected to 192.168.1.60 (192.168.1.60) port 19532
> GET / HTTP/1.1
> Host: 192.168.1.60:19532
> User-Agent: curl/8.5.0
> Accept: */*
>
< HTTP/1.1 406 Not Acceptable
< Date: Mon, 09 Jun 2025 23:57:17 GMT
< Connection: close
< Content-Type: text/plain
< Content-Length: 20
<
Unsupported method.
* Closing connection
----

If the server is not responding, make sure the service is running and that the port is open.
This may require opening port `19532` on the logging server firewall.

----
systemctl status systemd-journal-remote
